HTML v/s HTML5

HTML HTML5
The original HTML versions (HTML 4, XHTML 1.0) had a more limited set of elements and features. Introduces new elements like <header>, <nav>, <article>, <section>, <footer>, <canvas>, <audio>, <video>, <progress>, and more, providing better semantic structure and support for multimedia.
Required external plugins (like Adobe Flash) for multimedia content. Native support for <audio> and <video> elements, eliminating the need for third-party plugins. This simplifies multimedia integration.
Limited form validation and required additional scripting for advanced features. Provides built-in form validation, new input types (email, url, tel, number, etc.), and the <datalist> element for better user input handling.
Less emphasis on semantic elements, leading to a less expressive and structured document. Places a stronger emphasis on semantic elements, making it easier to create well-structured and meaningful web pages.
Basic support for scripting with JavaScript. Introduces new APIs like the Geolocation API, Web Storage API, Web Workers API, and others, enhancing the capabilities of web applications.
